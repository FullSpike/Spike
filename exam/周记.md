#关于maven的工具的简述和实现方法
    maven可以让不同的ide能运行相同的java代码，是一个被广泛认可的工具
        通过标准化的项目结构大幅简化了 Java 项目的编译、测试、打包和部署流程
        一般项目通过maven工具搭建的时候，会产生一个pom核心配置文件
        里面包含
        1、坐标 GroupId（组织标识）+ ArtifactId（项目名）+ Version（版本），唯一标识一个项目或依赖
        前者groupid多写入的是公司域名反写
        在dependency标签下可以选择导入的依赖，提供版本选择，通过坐标的形式载入
        2、生命周期（Lifecycle） 是预定义的构建执行流程
        clean → compile → test → package → install → deploy
        其中package可以将项目导成jar包
    一般ide已经自动加载ide的环境，当然也可以加载自己的maven环境，同时记得更改本地仓库


#关于mybatis框架的简述和实现方法
    mybatis时一个持久层框架，相当于半个项目，是一个项目模板
        它支持自定义 SQL、存储过程以及高级映射。MyBatis 免除了几乎所有的 JDBC 代码以及设置参数和获取结果集的工作
            同样里面有一个核心配置文件.xml，里面记录数据驱动，url，数据库的用户名和密码
            在映射mapper中，写入需要执行的spl语句，而且如果需要解决硬编码的问题可以同过mapper代理的方式解决
            即定义一个同名接口，在接口写入方法，在mapper用id与其对应起来，需要注意的是mapper和接口需要放在同一个文件夹下
            并且，mapper名通常为xxxMpaper.xml，接口名为xxx.mapper
        在导入mybatis依赖就可以运行该框架，也可以导入mybatis jar包
#关于项目规范三层架构
    分别为
    DAO层 service层 control层
    dao层，数据访问对象层，一般mapper接口放在这里，即一般关心数据的搬运，在数据库的增删改查，不负责业务的处理
